# 《操作系统》综合实验要求

实验题目：操作系统原理可视化实验展示平台设计与实现

适用专业：人工智能

实验学时：建议4-6周

实验方式：小组协作（2-3人/组）

# 一、实验目的

操作系统课程是人工智能与计算机相关专业的重要核心课程之一，是理解计算机系统工作机制和高层应用运行逻辑的关键环节。本综合实验旨在通过项目化与可视化结合的方式，促使学生将理论知识转化为工程实现能力，在“做中学”中深入理解操作系统的内核机制。

通过本综合实验，学生应能够：

1. 系统掌握操作系统的核心概念与机制：包括进程与线程的创建、通信与同步、CPU调度策略等。  
2. 强化编程与系统实现能力：能够独立设计并实现一个可运行的实验平台或演示软件，展示操作系统核心机制的动态运行过程。  
3. 培养团队协作与工程实践意识：通过小组分工、版本协作、模块整合和汇报展示，体验工程开发全流程。  
4. 提升可视化表达与创新思维：以交互化、图形化方式展示抽象的操作系统原理，提升系统可理解性与学习趣味性。  
5. 掌握跨平台与系统接口编程：能够在不同平台（Windows、Linux、FreeRTOS 等）上灵活实现进程管理与调度。

# 二、实验任务与总体要求

小组需在规定时间内完成一个可运行的操作系统实验展示软件，以及一份说明与展示PPT。整体任务如下：

# （一）核心功能模块（必做部分）

每个小组的软件系统需至少包含以下四个核心模块，均要求实现基础逻辑、提供动态可视化展示，并配有基础交互功能（如按钮、滑条、进度条、动画展示等）：

# 1. 进程与线程的创建与管理

能够模拟或真实调用系统接口创建多个进程/线程。  

展示进程的状态变化（创建、就绪、运行、阻塞、终止）及其转换关系。  

可视化设计建议：使用状态机图、动态动画或队列结构展示线程/进程切换。

# 2. 进程间通信（IPC）机制展示

要求至少实现一种通信方式 (可选：管道、共享内存、消息队列、Socket 等)。

软件中应通过可视化形式显示通信数据流 (如生产者一消费者模型、消息缓冲队列等) 。

鼓励通过不同颜色或动画效果展示数据传输方向与时序。

# 3. 基于信号量的进程同步机制

要求实现典型的同步模型（如生产者—消费者、读者—写者、哲学家就餐问题等）

展示信号量 P/V 操作的动态过程及其对线程阻塞/唤醒的影响。

可视化建议：实时显示信号量取值变化、阻塞队列状态和资源占用情况。

展示信号量 P/V 操作的动态过程及其对线程阻塞/唤醒的影响。

# 4.CPU调度算法展示与比较

要求至少实现三种调度算法：

(1) 先来先服务 (FCFS)  

(2）时间片轮转（RR）  

(3) 优先级或最短作业优先 (SJF)

可视化建议：

展示每个进程的到达时间、执行时间、优先级等参数； 

使用甘特图动态播放调度过程；

输出平均等待时间、周转时间等性能指标。

# （二）扩展与创新功能（自选加分部分）

在完成基础功能后，鼓励小组根据兴趣与特长进行创新性扩展，示例包括但不限于：

- 图形化操作系统“任务管理器”界面；  
- 动态内存分配与页面置换算法演示；  
- 模拟多核CPU并行调度；  
- 实时操作系统（RTOS）任务切换可视化；  
- 基于Web前端展示（如使用Python + Flask、Electron、Qt、Unity、或WebAssembly）。

# 三、实验成果与提交内容

每个小组须提交以下内容：

# (1) 实验展示软件源工程

- 完整的可编译、可运行的工程文件；  
- 含必要的文档说明（如README，环境依赖、运行方式）；  
- 可运行演示时需具备清晰的操作界面或命令行可视化输出。

# (2) 展示与汇报 PPT

PPT需系统说明软件的设计思路、实现原理、演示功能及创新点，推荐结构如下：

1. 选题背景与实验目的；  
2. 系统总体架构与模块划分；  
3. 各模块的设计原理与实现方法（含算法说明与可视化展示截图）；  
4. 功能展示与运行结果；  
5. 小组分工与项目管理（含版本控制方式）；  
6. 问题与改进方向。
